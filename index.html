üìã Core Database Schema Explanation

This ERD represents the foundational database structure for the SKOLARIS system, establishing the core relationships between campuses, users, roles, and permissions. The design follows a hierarchical multi-campus architecture that supports role-based access control (RBAC).

  • üè¢ CAMPUSES: Central entity managing multiple ICCT campus locations with unique identifiers and contact information
  • üë• USERS: Base user table linked to campuses, containing authentication and profile data for all system users
  • üîê ROLES: Defines 8 user roles (Super Admin, System Admin, Campus Admin, Registrar, Faculty, Staff, Student, Guest) with campus-specific or global scope
  • üîë PERMISSIONS: Granular permission system controlling access to specific modules and actions
  • üîó ROLE_PERMISSIONS: Many-to-many relationship mapping roles to their specific permissions
  • üë§ USER_ROLES: Junction table assigning roles to users within specific campus contexts

This schema ensures secure, scalable multi-campus management with fine-grained access control and data isolation between campuses.

üîó Core Schema Relationships
üè¢ ‚Üí üë• CAMPUSES to USERS

One-to-Many: Each campus can have multiple users, but each user belongs to exactly one campus. This ensures data isolation and campus-specific user management.

üè¢ ‚Üí üîê CAMPUSES to ROLES

One-to-Many: Each campus can define its own roles (campus-specific) or use global roles. Enables flexible role management per campus.

üîê ‚Üí üîë ROLES to PERMISSIONS

Many-to-Many: Through ROLE_PERMISSIONS junction table. Each role can have multiple permissions, and each permission can be assigned to multiple roles.

üë• ‚Üí üîê USERS to ROLES

Many-to-Many: Through USER_ROLES junction table. Users can have multiple roles within specific campus contexts, enabling flexible access control.
